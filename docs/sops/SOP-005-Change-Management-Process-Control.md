## 1. Purpose

To establish procedures for planning, evaluating, approving, implementing, and verifying changes to products, processes, procedures, equipment, and materials. This ensures changes are properly assessed for impact, approved at appropriate levels, and validated to maintain quality and compliance.

---

## 2. Scope

Changes covered include:
- Product design or specification changes
- Manufacturing process changes
- Procedure and SOP updates
- Equipment modifications or upgrades
- Supplier or material changes
- Tool or fixture changes
- Software/system changes
- Layout or facility changes

**Applies to:** All departments, all product and process changes

## 5. Change Request Process

### 5.1 Change Evaluation

Before any change:

1. **Identify Change Need**
   - What needs to change?
   - Why change (problem, improvement, regulatory)?
   - What's the benefit?

2. **Assess Impact**
   - Affected products/processes
   - Equipment/tools needed
   - Training requirements
   - Cost implications
   - Schedule/timeline

3. **Risk Analysis**
   - Quality risks (defect potential)
   - Safety risks
   - Regulatory risks
   - Operational risks
   - Financial risks

4. **Develop Change Plan**
   - Detailed steps to implement
   - Who will do it
   - Timeline and milestones
   - Resources needed
   - Verification method

## 7. Change Implementation

### 7.1 Pre-Implementation Checklist

Before implementing change:
- ✅ All approvals obtained
- ✅ Updated procedures written and reviewed
- ✅ Personnel trained and competent
- ✅ Equipment tested/verified
- ✅ Materials/parts available
- ✅ Backup plan if change fails
- ✅ Communication plan ready

### 7.2 Implementation Steps

1. **Preparation** (1-2 weeks before)
   - Train affected personnel
   - Procure materials/equipment
   - Test new procedures
   - Prepare backup plans

2. **Execution** (scheduled date)
   - Follow approved change plan
   - Document actual vs. planned
   - Monitor closely
   - Communicate status

3. **Verification** (immediately after)
   - First articles inspection
   - Process capability study
   - Quality checks
   - Performance verification

4. **Stabilization** (30-60 days)
   - Monitor performance metrics
   - Collect feedback
   - Adjust if needed
   - Verify training effectiveness

## 8. Documentation & Communication

### 8.1 Required Documentation

Document:
- Change request and approval
- Updated procedures (SOPs/WIs)
- Training records
- Implementation checklist
- Verification results
- Performance data before/after

### 8.2 Communication Plan

Communicate change to:
- **Production team:** What changes and why
- **Quality:** Inspection requirements
- **Supervisors:** Implementation timeline
- **Customers:** If product-related change
- **Regulators:** If regulatory compliance change

---

## 9. Change Verification & Validation

### 9.1 Verification Activities

| Change Type | Verification Method | Frequency |
| :--- | :--- | :--- |
| Product change | First article inspection | Before production |
| Process change | Process capability study | First 30 units |
| Equipment change | Performance test | After installation |
| Procedure change | Audit/observation | Within 2 weeks |
| Training change | Competency assessment | After training |

### 9.2 Performance Metrics Comparison

Track metrics before and after change:
- Quality (defect rate, scrap)
- Productivity (cycle time, yield)
- Safety (incidents, near-misses)
- Cost (material, labor)
- Customer satisfaction

---

## 10. Change Control System

### 10.1 Change Log

Maintain record of:
- Change ID number
- Date initiated/approved/implemented
- Description
- Status (pending, approved, implemented, closed)
- Implementation date
- Performance results

- Obtain approval if required
- Update data plate if applicable
- Update compliance records
